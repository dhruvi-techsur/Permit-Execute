**ReviewService** (`review.service.ts`):

Inject: `@InjectRepository(PermitApplication)`, `@InjectRepository(PermitStatusHistory)`, `@InjectRepository(Document)`, `S3Service`, `PermitLifecycleService`, `NotificationsService`, `AuditService`.

Methods:

`listApplications(reviewerId: string, query: ReviewerQueryDto)`:
- Query permit_applications WHERE status != 'draft' (reviewers see all non-draft)
- If query.assignedToMe === 'true': add WHERE reviewer_id = reviewerId
- Apply status/permitType filters, paginate, sort
- Return `{ data: PermitApplication[], meta: { total, page, limit } }`

`getApplicationDetail(applicationId: string, reviewerId: string)`:
- Find permit by id, throw NotFoundException if not found
- Load documents WHERE application_id = applicationId AND status = 'uploaded'
- For each document, generate presigned download URL (5 min) via S3Service.generateDownloadUrl()
- Load status history ordered by created_at DESC
- Return `{ application: PermitApplication, documents: DocumentWithDownloadUrl[], history: PermitStatusHistory[] }`

`assignReviewer(applicationId: string, reviewerId: string)`:
- Find permit, throw NotFoundException if not found
- If permit.reviewerId already set and !== reviewerId: throw ConflictException('Application already assigned to another reviewer')
- Update reviewer_id = reviewerId
- Write audit log: action='review.assigned', actorId=reviewerId, applicationId
- Return updated permit

`advanceToReview(applicationId: string, reviewerId: string)`:
- Find permit, throw 404 if not found
- Call lifecycleService.validateTransition(permit.status, PermitStatus.UNDER_REVIEW) — throws 422 if invalid
- Update status = 'under_review', reviewer_id = reviewerId (auto-assign if not assigned)
- Insert permit_status_history: { applicationId, fromStatus: permit.status, toStatus: 'under_review', changedBy: reviewerId }
- Create notification for permit.applicantId: type='status_change', title='Application Under Review', body=`Your application "${permit.title}" is now under review.`
- Write audit log: action='review.advanced', changedBy=reviewerId
- Return updated permit

`requestInfo(applicationId: string, reviewerId: string, dto: RequestInfoDto)`:
- Find permit, validate reviewer_id === reviewerId (403 if not their application unless they are admin)
- Call lifecycleService.validateTransition(permit.status, PermitStatus.ADDITIONAL_INFO_NEEDED)
- Update status = 'additional_info_needed'
- Insert permit_status_history: { fromStatus: permit.status, toStatus: 'additional_info_needed', changedBy: reviewerId, note: dto.reason }
- Create notification for permit.applicantId: type='info_request', title='Additional Information Required', body=`Your application "${permit.title}" requires additional information: ${dto.reason}`
- Write audit log: action='review.info_requested', metadata: { reason: dto.reason }
- Return updated permit

`respondToInfoRequest(applicationId: string, applicantId: string)`:
- Find permit, validate permit.applicantId === applicantId (403 if mismatch)
- Call lifecycleService.validateTransition(permit.status, PermitStatus.UNDER_REVIEW)
- Update status = 'under_review'
- Insert permit_status_history: { fromStatus: 'additional_info_needed', toStatus: 'under_review', changedBy: applicantId }
- Create notification for permit.reviewerId (if set): type='status_change', title='Application Re-submitted', body=`The applicant has responded to your information request for "${permit.title}".`
- Write audit log: action='review.info_responded', actorId=applicantId
- Return updated permit

`approveApplication(applicationId: string, reviewerId: string, dto: DecisionDto)`:
- Find permit, call lifecycleService.validateTransition(permit.status, PermitStatus.APPROVED)
- Update status = 'approved', decision_reason = dto.reason
- Insert permit_status_history
- Create notification for applicant: type='application_approved', title='Application Approved', body=`Your application "${permit.title}" has been approved. Reason: ${dto.reason}`
- Write audit log: action='review.approved'
- Return updated permit

`rejectApplication(applicationId: string, reviewerId: string, dto: DecisionDto)`:
- Find permit, call lifecycleService.validateTransition(permit.status, PermitStatus.REJECTED)
- Update status = 'rejected', decision_reason = dto.reason
- Insert permit_status_history
- Create notification for applicant: type='application_rejected', title='Application Rejected', body=`Your application "${permit.title}" has been rejected. Reason: ${dto.reason}`
- Write audit log: action='review.rejected'
- Return updated permit

**Ownership guard pattern:** For request-info, approve, reject — reviewers must be assigned (reviewer_id === reviewerId) OR have the admin role. Implement a private helper `ensureReviewerAccess(permit: PermitApplication, userId: string, userRole: string)` that checks this.

**ReviewController** (`review.controller.ts`):

```typescript
@Controller('api/v1/review')
@UseGuards(JwtAuthGuard)
export class ReviewController {
  constructor(private readonly reviewService: ReviewService) {}

  // REVIEWER + ADMIN only endpoints
  @Get('applications')
  @Roles('reviewer', 'admin')
  @UseGuards(RolesGuard)
  listApplications(@Query() query: ReviewerQueryDto, @Request() req) {
    return this.reviewService.listApplications(req.user.id, query);
  }

  @Get('applications/:id')
  @Roles('reviewer', 'admin')
  @UseGuards(RolesGuard)
  getDetail(@Param('id', ParseUUIDPipe) id: string, @Request() req) {
    return this.reviewService.getApplicationDetail(id, req.user.id);
  }

  @Post('applications/:id/assign')
  @Roles('reviewer', 'admin')
  @UseGuards(RolesGuard)
  assign(@Param('id', ParseUUIDPipe) id: string, @Request() req) {
    return this.reviewService.assignReviewer(id, req.user.id);
  }

  @Post('applications/:id/advance')
  @Roles('reviewer', 'admin')
  @UseGuards(RolesGuard)
  advance(@Param('id', ParseUUIDPipe) id: string, @Request() req) {
    return this.reviewService.advanceToReview(id, req.user.id);
  }

  @Post('applications/:id/request-info')
  @Roles('reviewer', 'admin')
  @UseGuards(RolesGuard)
  requestInfo(@Param('id', ParseUUIDPipe) id: string, @Body() dto: RequestInfoDto, @Request() req) {
    return this.reviewService.requestInfo(id, req.user.id, dto);
  }

  // APPLICANT only — respond to info request (NOT reviewer)
  @Post('applications/:id/respond')
  @Roles('applicant')
  @UseGuards(RolesGuard)
  respond(@Param('id', ParseUUIDPipe) id: string, @Request() req) {
    return this.reviewService.respondToInfoRequest(id, req.user.id);
  }

  @Post('applications/:id/approve')
  @Roles('reviewer', 'admin')
  @UseGuards(RolesGuard)
  approve(@Param('id', ParseUUIDPipe) id: string, @Body() dto: DecisionDto, @Request() req) {
    return this.reviewService.approveApplication(id, req.user.id, dto);
  }

  @Post('applications/:id/reject')
  @Roles('reviewer', 'admin')
  @UseGuards(RolesGuard)
  reject(@Param('id', ParseUUIDPipe) id: string, @Body() dto: DecisionDto, @Request() req) {
    return this.reviewService.rejectApplication(id, req.user.id, dto);
  }
}
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client→API | Request body/params into review handlers (reviewer actions on permit data) |
| API→notifications | Status change triggers notification creation for other users |
| DB→response | Full application detail (including document download URLs) returned to reviewer |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-04 | Elevation of privilege | GET /review/applications | mitigate | `@Roles('reviewer', 'admin') @UseGuards(RolesGuard)` on all reviewer endpoints — applicants cannot access the review list; enforced at controller level via JWT payload's `role` claim |
| T-03-05 | Elevation of privilege | POST /review/applications/:id/respond | mitigate | `@Roles('applicant') @UseGuards(RolesGuard)` restricts respond to applicant role; additionally `review.service.ts::respondToInfoRequest` checks `permit.applicantId === applicantId` — only the owning applicant can respond |
| T-03-06 | Elevation of privilege | request-info / approve / reject on unassigned permit | mitigate | `review.service.ts::ensureReviewerAccess` throws `ForbiddenException` when `permit.reviewerId !== reviewerId` and role is not admin — prevents reviewer A from acting on reviewer B's application |
| T-03-07 | Tampering | decision_reason content | mitigate | `DecisionDto` uses `@MinLength(10) @MaxLength(2000)` — empty or excessively long reasons are rejected at DTO validation before any DB write |
| T-03-08 | Tampering | Invalid state transitions via API | mitigate | `PermitLifecycleService.validateTransition` throws `UnprocessableEntityException` before any status update — server-side state machine cannot be bypassed by crafting request bodies |
| T-03-09 | Repudiation | All status transitions | mitigate | Every transition in `review.service.ts` calls `auditService.createEntry()` with `actorId`, `applicationId`, `action`, and relevant metadata — immutable audit trail for all reviewer decisions |
| T-03-10 | Information disclosure | Document download URLs in detail response | mitigate | `review.service.ts::getApplicationDetail` generates presigned URLs with 5-minute expiry via S3Service.generateDownloadUrl(expiresIn: 300) — leaked URLs expire quickly |
</threat_model>
