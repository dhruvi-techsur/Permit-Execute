  <action>
Create `backend/src/database/migrations/003_phase3_tables.ts` as a TypeORM migration. Follow the exact same pattern as migrations 001 and 002 — use `MigrationInterface` + `QueryRunner`.

**Step 1: Create the message_type enum:**
```sql
CREATE TYPE message_type AS ENUM ('applicant_message', 'reviewer_message', 'status_note', 'info_request')
```

**Step 2: Create the messages table:**
```sql
CREATE TABLE messages (
    id              UUID            PRIMARY KEY DEFAULT gen_random_uuid(),
    application_id  UUID            NOT NULL REFERENCES permit_applications(id) ON DELETE CASCADE,
    sender_id       UUID            NOT NULL REFERENCES users(id) ON DELETE RESTRICT,
    body            TEXT            NOT NULL,
    message_type    message_type    NOT NULL,
    has_attachment  BOOLEAN         NOT NULL DEFAULT FALSE,
    created_at      TIMESTAMPTZ     NOT NULL DEFAULT NOW(),
    updated_at      TIMESTAMPTZ     NOT NULL DEFAULT NOW()
);
CREATE INDEX idx_messages_application ON messages (application_id, created_at ASC);
CREATE INDEX idx_messages_sender      ON messages (sender_id);
CREATE INDEX idx_messages_type        ON messages (message_type);
```

Apply the existing `trigger_set_updated_at()` trigger to messages:
```sql
CREATE TRIGGER set_updated_at
  BEFORE UPDATE ON messages
  FOR EACH ROW EXECUTE FUNCTION trigger_set_updated_at()
```

**Step 3: Create the message_attachments table:**
```sql
CREATE TABLE message_attachments (
    id              UUID        PRIMARY KEY DEFAULT gen_random_uuid(),
    message_id      UUID        NOT NULL REFERENCES messages(id) ON DELETE CASCADE,
    file_name       TEXT        NOT NULL,
    storage_key     TEXT        NOT NULL UNIQUE,
    file_type       TEXT        NOT NULL,
    file_size_bytes BIGINT      NOT NULL,
    created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
CREATE INDEX idx_msg_attach_message ON message_attachments (message_id);
CREATE INDEX idx_msg_attach_key     ON message_attachments (storage_key);
```

**Step 4: Implement down() in correct reverse order:**
1. DROP TRIGGER set_updated_at ON messages (IF EXISTS)
2. DROP TABLE IF EXISTS message_attachments CASCADE
3. DROP TABLE IF EXISTS messages CASCADE
4. DROP TYPE IF EXISTS message_type CASCADE

Use `await queryRunner.query(sql)` for each statement. Name the migration class `Phase3Tables1721000000003` with `name = 'Phase3Tables1721000000003'`.

**Important:** The reviewer workflow (STAT-03/04/05/06) does NOT need new tables. The `permit_applications.reviewer_id` column (already created in migration 002) is used for reviewer assignment. Status transitions write to `permit_status_history` (already exists). Approval/rejection writes `decision_reason` to `permit_applications` (column already exists). This migration is purely for messaging infrastructure.

Also update the seed file (`backend/src/database/seeds/seed.ts`) to add 2-3 sample messages to a seed permit application so UAT testers have a message thread to interact with immediately:
- Find or create the first submitted permit application for the applicant seed user
- Insert 2 messages: one from applicant, one from reviewer
- Use `ON CONFLICT (id) DO NOTHING` to keep it idempotent
