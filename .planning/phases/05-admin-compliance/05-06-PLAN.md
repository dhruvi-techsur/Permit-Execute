**Step 1: Fix aria-labels on icon-only buttons across all feature pages**

Search for buttons that contain only an icon (SVG, Lucide icon component) with NO visible text label. Each MUST have `aria-label`. Patterns to find and fix:

```tsx
// BAD — no accessible name:
<button onClick={...}><ChevronRight /></button>

// GOOD — aria-label describes the action:
<button aria-label="View permit details" onClick={...}><ChevronRight /></button>
```

Apply to ALL pages: PermitListPage, PermitDetailPage, ReviewListPage, ReviewDetailPage, DashboardPage, and all three admin pages. Common icon-only buttons to fix:
- Navigation arrows (next/previous page) → `aria-label="Next page"` / `aria-label="Previous page"`
- Close/dismiss buttons on toasts, modals → `aria-label="Close"`
- Download buttons → `aria-label="Download document {filename}"`
- Upload buttons → `aria-label="Upload document"`
- Expand/collapse buttons → `aria-label="Expand details"` / `aria-label="Collapse details"`
- Notification bell icon → `aria-label="Notifications"`

**Step 2: Fix loading skeletons (WCAG 4.1.3 Status Messages)**

Find all skeleton/loading components or inline skeleton JSX across all pages. Add:
```tsx
<div role="status" aria-live="polite" aria-label="Loading content">
  {/* skeleton content */}
</div>
```

If loading state is a boolean and the component conditionally renders skeleton vs content, ensure the skeleton wrapper has `role="status"`.

**Step 3: Fix keyboard navigation on interactive cards/rows**

Check PermitListPage, ReviewListPage — if permit/application cards/rows are clickable divs (`<div onClick={...}>`), convert them to:
```tsx
<div
  role="button"
  tabIndex={0}
  onClick={handleClick}
  onKeyDown={(e) => { if (e.key === 'Enter' || e.key === ' ') handleClick(); }}
  aria-label="View permit {permit.title}"
>
```
Or better: wrap in `<a href={`/permits/${permit.id}`}>` which is natively keyboard-operable.

**Step 4: Add aria-live regions for dynamic content**

In DashboardPage, any notification count badge or status update that changes dynamically should have `aria-live="polite"` on its container. For example: `<span aria-live="polite" aria-atomic="true">{unreadCount}</span>` for unread message counts.
