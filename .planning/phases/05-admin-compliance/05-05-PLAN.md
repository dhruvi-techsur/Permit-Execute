**Step 1: Install @axe-core/playwright**

```bash
cd frontend && npm install --save-dev @axe-core/playwright
```

**Step 2: Create `frontend/src/components/ui/SkipNav.tsx`**:

```typescript
export function SkipNav() {
  return (
    <a
      href="#main-content"
      className="sr-only focus:not-sr-only focus:absolute focus:top-4 focus:left-4 focus:z-50 focus:px-4 focus:py-2 focus:bg-white focus:text-blue-700 focus:border focus:border-blue-700 focus:rounded focus:shadow-lg focus:outline-none focus:ring-2 focus:ring-blue-500"
    >
      Skip to main content
    </a>
  );
}
```

The `sr-only` class hides it from sighted users. `focus:not-sr-only` reveals it when focused via keyboard Tab. This covers WCAG 2.4.1 Bypass Blocks.

**Step 3: Update `frontend/src/layout/AppShell.tsx`**:

Find the root container element (typically the outermost `<div>` or `<main>`). Make these changes:
1. Import `SkipNav` from `../components/ui/SkipNav`.
2. Add `<SkipNav />` as the FIRST child of the root container, BEFORE the Sidebar.
3. Find the main content area (the area that renders children/Outlet). Add `id="main-content"` and `tabIndex={-1}` to it. Example: `<main id="main-content" tabIndex={-1} className="flex-1 overflow-auto p-6">`.
4. Add `role="navigation" aria-label="Main navigation"` to the `<nav>` element wrapping the Sidebar (if not already present).

**Step 4: Fix modals for focus trap + Escape key (WCAG 2.1.1, 2.4.3)**

Check `frontend/src/components/ui/Modal.tsx` (or whichever shared modal component exists). Apply these fixes:
1. Add `role="dialog"` and `aria-modal="true"` to the modal container div.
2. Add `aria-labelledby` pointing to the modal's heading id.
3. Add focus trap: when modal renders, focus moves to first focusable element inside. Use `useEffect` + `ref` to query `button, [href], input, select, textarea, [tabindex]:not([tabindex="-1"])` and call `.focus()` on the first result.
4. Add Escape key handler: `useEffect` adds `keydown` listener → if `event.key === 'Escape'`, calls `onClose()`. Returns cleanup.
5. Add backdrop `aria-hidden="true"` on the backdrop overlay div (prevents screen readers from reading the blurred background).

Apply the same focus trap + Escape + aria attributes to `StatusActionPanel.tsx` if it renders as a modal/dialog (check if it uses `fixed`/`absolute` positioning as an overlay).

**Step 5: Fix auth form accessibility (WCAG 4.1.2, 1.3.1)**

For `LoginPage.tsx`, `RegisterPage.tsx`, `ForgotPasswordPage.tsx`, `ResetPasswordPage.tsx`:

Each `<input>` MUST have a corresponding `<label htmlFor="{inputId}">`. Do NOT use placeholder as the only label. Ensure:
1. Each input has a unique `id` attribute.
2. Each input has a matching `<label htmlFor="{id}">` above it (or use `aria-label` if label is visually hidden by design).
3. Error messages shown below inputs have `role="alert"` and are associated with the input via `aria-describedby="{errorId}"` on the input.

Example pattern:
```tsx
<label htmlFor="login-email" className="block text-sm font-medium text-gray-700 mb-1">
  Email address
</label>
<input
  id="login-email"
  type="email"
  autoComplete="email"
  aria-required="true"
  aria-describedby={emailError ? "login-email-error" : undefined}
  className="..."
/>
{emailError && (
  <p id="login-email-error" role="alert" className="text-red-600 text-sm mt-1">{emailError}</p>
)}
```

Apply to ALL form inputs in all four auth pages.
