**PermitLifecycleService** (`permit-lifecycle.service.ts`):
```typescript
@Injectable()
export class PermitLifecycleService {
  private readonly VALID_TRANSITIONS: Record<PermitStatus, PermitStatus[]> = {
    [PermitStatus.DRAFT]: [PermitStatus.SUBMITTED],
    [PermitStatus.SUBMITTED]: [PermitStatus.UNDER_REVIEW],
    [PermitStatus.UNDER_REVIEW]: [PermitStatus.ADDITIONAL_INFO_NEEDED, PermitStatus.APPROVED, PermitStatus.REJECTED],
    [PermitStatus.ADDITIONAL_INFO_NEEDED]: [PermitStatus.UNDER_REVIEW],
    [PermitStatus.APPROVED]: [],
    [PermitStatus.REJECTED]: [],
  };

  validateTransition(from: PermitStatus, to: PermitStatus): void {
    if (!this.VALID_TRANSITIONS[from]?.includes(to)) {
      throw new UnprocessableEntityException(
        `Invalid status transition: ${from} → ${to}. Allowed: ${this.VALID_TRANSITIONS[from]?.join(', ') || 'none'}`
      );
    }
  }
}
```

**AuditService** (`audit.service.ts`):
```typescript
@Injectable()
export class AuditService {
  constructor(@InjectRepository(AuditLog) private readonly auditRepo: Repository<AuditLog>) {}

  async createEntry(
    actorId: string,
    applicationId: string | null,
    action: string,
    metadata: Record<string, unknown> = {},
  ): Promise<void> {
    // Strip sensitive keys before storing
    const safe = { ...metadata };
    ['password', 'token', 'secret', 'key'].forEach(k => delete safe[k]);
    await this.auditRepo.insert({ actorId, applicationId, action, metadata: safe });
  }
}
```

**PermitsService** (`permits.service.ts`) — Implement all business logic using TypeORM repository:

Inject: `@InjectRepository(PermitApplication)`, `@InjectRepository(PermitStatusHistory)`, `PermitLifecycleService`, `AuditService`.

Methods:
- `create(userId: string, dto: CreatePermitDto): Promise<PermitApplication>` — Insert with status='draft', applicantId=userId, write audit log 'permit.created'
- `findAll(userId: string, query: PermitListQueryDto): Promise<{ data: PermitApplication[]; meta: { total: number; page: number; limit: number } }>` — Filter by applicantId=userId (applicants see only own), apply status/type filters, paginate, sort by column map (createdAt→created_at, updatedAt→updated_at, submissionDate→submission_date)
- `findOne(id: string, userId: string): Promise<PermitApplication>` — Find by id, throw `ForbiddenException` if applicantId !== userId (and user is not admin/reviewer), throw `NotFoundException` if not found
- `update(id: string, userId: string, dto: UpdatePermitDto): Promise<PermitApplication>` — Throw `BadRequestException` if status !== 'draft'; update fields; write audit log 'permit.updated'
- `submit(id: string, userId: string): Promise<PermitApplication>` — validate transition draft→submitted via lifecycle service; check at least 1 document in 'uploaded' status exists (query documents table); set submissionDate=NOW(); write history row (fromStatus='draft', toStatus='submitted', changedBy=userId); write audit log 'permit.submitted'
- `getHistory(id: string, userId: string): Promise<PermitStatusHistory[]>` — Verify ownership first; return history ordered by created_at DESC

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-02 | Elevation of privilege | `GET /api/v1/permits/:id` | mitigate | `permits.service.ts::findOne` throws `ForbiddenException` when `permit.applicantId !== userId` — ownership check at the service layer before any data is returned |
| T-02-03 | Tampering | `PATCH /api/v1/permits/:id` | mitigate | `permits.service.ts::update` throws `BadRequestException` when `status !== 'draft'` — prevents modification of submitted/approved/rejected applications |
| T-02-04 | Elevation of privilege | `POST /api/v1/permits/:id/submit` | mitigate | `PermitLifecycleService.validateTransition` throws `UnprocessableEntityException` for any invalid transition — server-side state machine cannot be bypassed by client |
| T-02-05 | Information disclosure | GET /api/v1/permits list | mitigate | `permits.service.ts::findAll` always filters `WHERE applicant_id = $userId` — no applicant can enumerate other users' applications |
| T-02-06 | Repudiation | Status transitions | mitigate | Every status change writes `permit_status_history` row AND `audit_logs` entry with `changedBy` and `actorId` — immutable record of who changed what and when |
| T-02-07 | Tampering | audit_logs.metadata | mitigate | `audit.service.ts::createEntry` strips `password`, `token`, `secret`, `key` keys before insert — no credential leakage into audit trail |
</threat_model>
