**S3Service** (`s3.service.ts`) — AWS SDK v3 wrapper for MinIO:

Install if not present: `@aws-sdk/client-s3`, `@aws-sdk/s3-request-presigner`.

```typescript
@Injectable()
export class S3Service implements OnModuleInit {
  private client: S3Client;
  private readonly bucket = 'permit-documents';

  onModuleInit() {
    this.client = new S3Client({
      endpoint: process.env.MINIO_ENDPOINT || 'http://minio:9000',
      region: 'us-east-1',
      credentials: {
        accessKeyId: process.env.MINIO_ROOT_USER || 'minioadmin',
        secretAccessKey: process.env.MINIO_ROOT_PASSWORD || 'minioadmin',
      },
      forcePathStyle: true, // Required for MinIO
    });
    // Ensure bucket exists on startup
    this.ensureBucket().catch(err => console.warn('MinIO bucket check failed:', err.message));
  }

  private async ensureBucket(): Promise<void> {
    try {
      await this.client.send(new HeadBucketCommand({ Bucket: this.bucket }));
    } catch {
      await this.client.send(new CreateBucketCommand({ Bucket: this.bucket }));
    }
  }

  async generateUploadUrl(storageKey: string, mimeType: string, expiresIn = 900): Promise<string> {
    const cmd = new PutObjectCommand({ Bucket: this.bucket, Key: storageKey, ContentType: mimeType });
    return getSignedUrl(this.client, cmd, { expiresIn });
  }

  async generateDownloadUrl(storageKey: string, fileName: string, expiresIn = 300): Promise<string> {
    const cmd = new GetObjectCommand({
      Bucket: this.bucket,
      Key: storageKey,
      ResponseContentDisposition: `attachment; filename="${fileName}"`,
    });
    return getSignedUrl(this.client, cmd, { expiresIn });
  }

  async deleteObject(storageKey: string): Promise<void> {
    await this.client.send(new DeleteObjectCommand({ Bucket: this.bucket, Key: storageKey }));
  }
}
```

**DocumentsService** — key methods:

`presign(permitId, userId, dto)`:
1. Verify permit exists and applicantId === userId (throw 403 if not)
2. Validate mimeType and fileSizeBytes (400 if invalid — already done by DTO but double-check max-docs: if 20 docs already uploaded for this permit, throw 400 "maximum 20 documents reached")
3. Generate storageKey: `permit-documents/${permitId}/${uuidv4()}/${dto.fileName}`
4. Insert Document row with status='pending', storageKey, fileName, fileType=mimeType, fileSizeBytes, applicationId=permitId, uploadedBy=userId
5. Call `s3Service.generateUploadUrl(storageKey, mimeType, 900)`
6. Return `{ documentId, uploadUrl, storageKey }`

`confirmUpload(permitId, userId, dto)`:
1. Find document by dto.documentId + applicationId=permitId
2. Throw 404 if not found; 403 if uploadedBy !== userId
3. Throw 400 if status !== 'pending' (already confirmed)
4. Update status='uploaded'
5. Return updated Document

`listDocuments(permitId, userId)`:
1. Verify permit ownership
2. Return all documents WHERE applicationId=permitId AND status NOT IN ('deleted') ordered by createdAt ASC

`downloadUrl(permitId, docId, userId)`:
1. Verify permit ownership; find document by docId+applicationId=permitId
2. Throw 404 if not found; 400 if status not 'uploaded'
3. Call `s3Service.generateDownloadUrl(storageKey, fileName, 300)`
4. Return `{ downloadUrl, fileName }`

`softDelete(permitId, docId, userId)`:
1. Verify permit ownership; find document
2. Update status='deleted'
3. (Do NOT delete from S3 immediately — soft-delete only for now)

**NotificationsService** — exact methods for TechArch endpoints:

```typescript
@Injectable()
export class NotificationsService {
  constructor(@InjectRepository(Notification) private readonly notifRepo: Repository<Notification>) {}

  // Called by other services (e.g. permits, Phase 3 workflow)
  async createNotification(
    userId: string,
    type: NotificationType,
    title: string,
    body: string,
    applicationId?: string,
  ): Promise<void> {
    await this.notifRepo.insert({ userId, type, title, body, applicationId: applicationId ?? null, isRead: false });
  }

  // GET /api/v1/notifications
  async findAll(userId: string, page = 1, limit = 20, unreadOnly = false) {
    const qb = this.notifRepo.createQueryBuilder('n')
      .where('n.user_id = :userId', { userId })
      .orderBy('n.created_at', 'DESC')
      .skip((page - 1) * limit)
      .take(limit);
    if (unreadOnly) qb.andWhere('n.is_read = false');
    const [data, total] = await qb.getManyAndCount();
    return { data, meta: { total, page, limit } };
  }

  // GET /api/v1/notifications/unread-count
  async getUnreadCount(userId: string): Promise<{ count: number }> {
    const count = await this.notifRepo.count({ where: { userId, isRead: false } });
    return { count };
  }

  // PATCH /api/v1/notifications/:id/read
  async markRead(id: string, userId: string): Promise<void> {
    const notif = await this.notifRepo.findOne({ where: { id, userId } });
    if (!notif) throw new NotFoundException('Notification not found');
    await this.notifRepo.update(id, { isRead: true });
  }

  // PATCH /api/v1/notifications/read-all
  async markAllRead(userId: string): Promise<void> {
    await this.notifRepo.update({ userId, isRead: false }, { isRead: true });
  }
}
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client→API | presign request body (fileName, mimeType, fileSizeBytes) entering document handler |
| API→S3 | presigned URL generation — URL handed to client who PUTs directly to MinIO |
| client→MinIO | Direct S3 PUT using presigned URL (bypasses API) |
| DB→response | Document/notification data returned to authenticated user |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-08 | Tampering | `POST presign` mimeType | mitigate | `presign-request.dto.ts` uses `@IsIn(ALLOWED_MIME_TYPES)` — allowlist validation rejects non-PDF/JPEG/PNG/DOCX MIME types before S3 URL is generated |
| T-02-09 | Denial of service | `POST presign` fileSizeBytes | mitigate | `@Max(26_214_400)` on `fileSizeBytes` in PresignRequestDto — enforces 25MB limit server-side before presigned URL generation |
| T-02-10 | Elevation of privilege | document ownership | mitigate | `documents.service.ts::presign` and `softDelete` verify `permit.applicantId === userId` before any S3 or DB operation |
| T-02-11 | Information disclosure | presigned URL expiry | mitigate | Upload URLs expire in 15 minutes (`expiresIn: 900`), download URLs in 5 minutes (`expiresIn: 300`) — limits window for leaked URL exploitation |
| T-02-12 | Information disclosure | GET /notifications | mitigate | `notifications.service.ts::findAll` always filters `WHERE user_id = :userId` — users cannot enumerate other users' notifications |
| T-02-13 | Tampering | storageKey path | mitigate | `storageKey` is generated server-side as `permit-documents/${permitId}/${uuidv4()}/${fileName}` — client never controls the storage key, preventing path traversal into other permits' documents |
</threat_model>
